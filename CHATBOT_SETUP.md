# TxLogic AI Chatbot Setup Guide

## Overview

The TxLogic chatbot has been upgraded to use **Google Gemini API** for intelligent, dynamic responses instead of hardcoded/programmed data. It uses **MongoDB Atlas** for persistent chat history storage.

## Key Features

✅ **AI-Powered Responses**: Uses Google Gemini API for contextual, intelligent replies
✅ **No Hardcoded Data**: All responses are generated dynamically based on conversation context
✅ **MongoDB Integration**: Chat history is stored in MongoDB Atlas for persistence
✅ **Modern UI**: Clean, minimal design with gradient accents and smooth animations
✅ **Responsive Design**: Works on all screen sizes (mobile, tablet, desktop)
✅ **Dark/Light Mode**: Toggle between themes
✅ **Clear Chat**: Option to clear conversation history

## Prerequisites

1. **Node.js** (v16 or higher)
2. **MongoDB Atlas** account
3. **Google Gemini API** key

## Setup Instructions

### 1. Install Dependencies

```bash
npm install
```

### 2. Configure Environment Variables

Create a `.env` file in the root directory (copy from `.env.example`):

```bash
cp .env.example .env
```

Edit `.env` and add your credentials:

```env
# MongoDB Atlas Connection String
MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/txlogic?retryWrites=true&w=majority

# Google Gemini API Key
# Get your API key from: https://makersuite.google.com/app/apikey
GEMINI_API_KEY=your_gemini_api_key_here

# Server Port
PORT=5000

# Frontend API URL
VITE_API_URL=http://localhost:5000/api
```

### 3. Get Google Gemini API Key

1. Go to [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Sign in with your Google account
3. Click "Create API Key"
4. Copy the API key and paste it in your `.env` file

### 4. Start the Backend Server

```bash
cd server
node index.js
```

The server will start on `http://localhost:5000`

### 5. Start the Frontend (in a new terminal)

```bash
npm run dev
```

The frontend will start on `http://localhost:5173` (or your configured port)

## How It Works

### Architecture

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   React Frontend│────▶│  Express Server │────▶│  Google Gemini  │
│   (ChatBot.tsx) │     │  (server/index) │     │     API         │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │
        │                       ▼
        │               ┌─────────────────┐
        │               │  MongoDB Atlas  │
        │               │  (Chat History) │
        │               └─────────────────┘
        │
        ▼
┌─────────────────┐
│     User        │
└─────────────────┘
```

### Data Flow

1. **User sends a message** → Frontend captures input
2. **Frontend sends to backend** → POST `/api/chat`
3. **Backend processes with Gemini** → Sends conversation context + user message
4. **Gemini generates response** → Returns AI-powered reply
5. **Backend saves to MongoDB** → Stores chat history
6. **Frontend displays response** → Shows message with animations

### Clearing Programmed Data

The chatbot no longer uses programmed/hardcoded responses. All replies are generated by Gemini API based on:

- Current user message
- Conversation history
- User context (name, preferences)
- System instructions

To clear chat history:
- Click the **trash icon** in the chatbot header
- This removes all stored data from MongoDB and resets the conversation

## API Endpoints

### POST `/api/chat`
Get AI response from Gemini

**Request:**
```json
{
  "message": "How can I track my package?",
  "conversationHistory": [
    {"role": "user", "content": "Hello"},
    {"role": "assistant", "content": "Hi there!"}
  ],
  "userInfo": {
    "name": "John",
    "preferredTracking": "package"
  }
}
```

**Response:**
```json
{
  "response": "I can help you track your package! Please provide your tracking ID..."
}
```

### POST `/api/chat/history`
Save chat history to MongoDB

### GET `/api/chat/history/:sessionId`
Retrieve chat history

### DELETE `/api/chat/history/:sessionId`
Clear chat history

## Troubleshooting

### "Gemini API Error"
- Check your API key in `.env`
- Ensure you have internet connectivity
- Verify API key is valid at [Google AI Studio](https://makersuite.google.com)

### "MongoDB Connection Error"
- Verify your MongoDB connection string
- Check if your IP is whitelisted in MongoDB Atlas
- Ensure database user has correct permissions

### Chatbot not responding
- Make sure backend server is running on port 5000
- Check browser console for errors
- Verify `VITE_API_URL` in `.env` matches your backend URL

## Production Deployment

### Environment Variables for Production

```env
# Use production MongoDB connection
MONGO_URI=mongodb+srv://prod-user:prod-pass@cluster.mongodb.net/txlogic-prod

# Use production Gemini API key
GEMINI_API_KEY=prod_api_key

# Set production port
PORT=5000

# Set production API URL
VITE_API_URL=https://your-domain.com/api
```

### Build for Production

```bash
# Build frontend
npm run build

# Start backend in production
NODE_ENV=production node server/index.js
```
